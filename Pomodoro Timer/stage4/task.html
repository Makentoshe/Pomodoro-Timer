before we create a static UI, let's put a view on the screen while your app is working. To do that, we can create a view through the code: just use the addView method (https://startandroid.ru/ru/uroki/vse-uroki-spiskom/49-16-layoutparams.html). This approach is not very convenient: our code becomes dirty and high coupled. The best way to create a new view on the screen is getting it from xml, like Activity. The links above show how to do that.

AlertDialog (https://developer.android.com/guide/topics/ui/dialogs?hl=en): sometimes we want to show something in front of the screen; in this case, we can use AlertDialog. This thing appears in front of the screen; it can contain views, and we can set up the behavior of our app depending on events in this dialog box. The link describes this approach.

But the best and more correct way is to use a DialogFragment. It has the same functional as AlertDialog but it also handles any possible dialog states on his own. This allows us to do not to think about memory leaks or device rotations.

Description

In this stage, you should set up a stopwatch upper limit in seconds via DialogFragment with single EditText and default dialog buttons. When the time reaches the upper limit the timer color changes to red
Add Button with text "Settings" and id settingsButton. DialogFragment should appear after clicking this button. DialogFragment should have:

  1. EditText with id upperLimit
  2. Button with a text "OK"
  3. Button with a text "Cancel"

Bottom buttons can be placed by using AlertDialog.Builder methods setNegativeButton and setPositiveButton. You should use it and avoid create your own buttons with the same idea

If there is a number in EditText and the user clicks "OK", set the upper limit as the number of seconds. If a user clicks on "Cancel" button, only AlertDialog closes.
